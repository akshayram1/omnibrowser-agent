Docs

Everything you need to install, initialize, and run your first browser agent.

Installation

npm install @akshayram1/omnibrowser-agent

Quick Start

import { createBrowserAgent } from "@akshayram1/omnibrowser-agent";

const agent = createBrowserAgent(
  {
    goal: "Open CRM and find customer John Smith",
    mode: "human-approved",
    planner: { kind: "heuristic" }
  },
  {
    onStep: (result) => console.log(result.message),
    onApprovalRequired: (action) => console.log("Needs approval:", action),
    onDone: (result) => console.log("Done:", result.message),
    onMaxStepsReached: (session) => console.log("Max steps hit", session.history)
  }
);

await agent.start();

// Resume after approval:
await agent.resume();

// Inspect state:
console.log(agent.isRunning, agent.hasPendingAction);

// Stop:
agent.stop();

AbortSignal support

const controller = new AbortController();
const agent = createBrowserAgent({ goal: "...", signal: controller.signal });
agent.start();

// Cancel from outside:
controller.abort();

Execution Modes

human-approved

Requires explicit approval for sensitive actions. Best for production-like workflows.

autonomous

Runs actions continuously with fewer pauses. Best for rapid iteration and demos.

Planner Options

heuristic

Zero-dependency regex-based planner. Works offline. Best for simple, predictable goals.

webllm

Delegates to a local WebLLM bridge (window.__browserAgentWebLLM). Fully private, no API calls.

page-agent

Delegates to an alibaba/page-agent bridge (window.__browserAgentPageAgent). Best for complex multi-step goals.

Supported Actions

ActionDescription
clickClick an element by CSS selector
typeType text into an input or textarea
navigateNavigate to a URL
extractExtract text from an element
scrollScroll a container or the page
focusFocus an element (useful for dropdowns)
waitPause for a given number of milliseconds
doneSignal task completion

Safety Notes

  • Prefer scoped selectors for deterministic action targeting.
  • Use human-approved mode for workflows that mutate critical data.
  • Log onStep output for auditability and debugging.

Architecture

How OmniBrowser Agent is structured internally and how its components interact.

Goals

  • Local-first runtime in browser
  • Privacy-first defaults
  • Open-source composable planner/executor contracts
  • Human-approved mode for risky actions

Runtime Components

Popup UI

Starts/stops sessions. Picks execution mode (autonomous, human-approved) and planner (heuristic, webllm, page-agent).

Background Service Worker

Session state machine per tab. Tick loop orchestration and approval handling.

Content Agent

pageObserver — page snapshot extraction.
planner — next-action decision.
safety — risk gating.
executor — DOM action execution.

Action Contracts

All components share a typed action protocol defined in src/shared/contracts.ts:

  • click — click element by CSS selector
  • type — type text into input/textarea
  • navigate — navigate to URL
  • extract — extract text from element
  • scroll — scroll container or page
  • focus — focus an element
  • wait — pause for N milliseconds
  • done — signal task completion

Safety Model

  • Block invalid URL protocols
  • Review risky actions (submit/delete/pay-like selectors)
  • In human-approved mode, review-level actions require manual approval before execution

Planner Bridges

All planner bridges follow the same pattern — an object attached to window that implements a plan() method returning an AgentAction. The core library has zero runtime dependencies; bridge implementations are provided by the consumer.

WebLLM bridge

window.__browserAgentWebLLM = {
  async plan(input, modelId) {
    // call your local WebLLM engine and return one AgentAction
    return { type: "done", reason: "result from model" };
  }
};

page-agent bridge

import { PageAgent } from "page-agent";

const pa = new PageAgent({
  baseURL: "https://api.openai.com/v1",
  model: "gpt-4o",
  apiKey: "sk-..."
});

window.__browserAgentPageAgent = {
  async plan(input) {
    const result = await pa.execute(input.goal);
    return { type: "done", reason: result.data };
  }
};

Current Limitations

  • No persistent long-term memory yet
  • No task DSL or skills registry yet
  • Risk scoring is a simple keyword heuristic
  • No robust selector healing yet

Embedding Guide

How to embed OmniBrowser Agent as a library inside your own web application.

Install

npm install @akshayram1/omnibrowser-agent

Basic Usage

import { createBrowserAgent } from "@akshayram1/omnibrowser-agent";

const agent = createBrowserAgent(
  {
    goal: "Search contact Jane Doe and open profile",
    mode: "human-approved",
    planner: { kind: "heuristic" },
    maxSteps: 15,
    stepDelayMs: 400
  },
  {
    onStep: (result) => console.log("step", result),
    onApprovalRequired: (action) => {
      console.log("approval required", action);
      // Show your own modal/button then call approvePendingAction()
    },
    onDone: (result) => console.log("done", result),
    onError: (error) => console.error(error)
  }
);

await agent.start();

Approve a Pending Action

await agent.approvePendingAction();

Stop Running Session

agent.stop();

WebLLM Mode

To use planner mode webllm, provide a local bridge in your app:

window.__browserAgentWebLLM = {
  async plan(input, modelId) {
    // call your local WebLLM engine and return one AgentAction JSON
    return { type: "done", reason: `Implement bridge with model ${modelId ?? "default"}` };
  }
};

// Then configure:
planner: { kind: "webllm", modelId: "Llama-3.2-1B-Instruct-q4f16_1-MLC" }

page-agent Mode

Install page-agent and wire the bridge for complex multi-step goals: 

npm install page-agent
import { PageAgent } from "page-agent";

const pa = new PageAgent({
  baseURL: "https://api.openai.com/v1",
  model: "gpt-4o",
  apiKey: "sk-..."
});

window.__browserAgentPageAgent = {
  async plan(input) {
    const result = await pa.execute(input.goal);
    return { type: "done", reason: result.data };
  }
};

// Then configure:
planner: { kind: "page-agent" }

Notes

  • For production, mount this inside an authenticated app shell and add your own permission checks.
  • human-approved mode is recommended for CRM, finance, and admin actions.
  • page-agent is not bundled — it must be installed separately by the consumer.
